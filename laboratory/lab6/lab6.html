  1. ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

    2. Esto se denomina validación de formulario en el lado del cliente y ayuda a garantizar que los datos que se envían coinciden con los requisitos establecidos en los diversos controles de formulario.

  2. ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

    3. Es tan sencillo como introducir un par de palabras reservadas en el código HTML utilizando la consola del desarrollador de nuestro navegador.

  3. Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

    4. Aunque las validaciones del lado del cliente con JavaScript pueden ser eludidas por usuarios malintencionados, siguen siendo una buena práctica porque mejoran la experiencia del usuario al proporcionar retroalimentación inmediata y reducen la carga en el servidor al evitar el envío de datos incorrectos o mal formateados. Sin embargo, estas validaciones deben complementarse con validaciones en el servidor para garantizar la seguridad y la integridad de los datos.
