1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una red de computadoras que se encuentran interconectadas a nivel mundial para compartir información, en cambio, la Wold Wide Web es en un complejo sistema de hipertextos e hipermedios interconectados entre sí y a los que puede tenerse acceso mediante una conexión a Internet y un conjunto de software especializado.

2. ¿Cuáles son las partes de un URL?

Protocolo: Es el primer componente de una URL y determina cómo se transmiten los datos.Como HTTP, HTTPS, FTP

Dominio: Es la dirección del sitio web. Proporciona una forma fácil de recordar los sitios en lugar de usar direcciones IP.

Subdominio: Es una parte adicional del nombre de dominio que puede representar una sección específica del sitio. Generalmente se encuentra antes del nombre de dominio principal.

Ruta: Indica la ubicación específica de un recurso en el servidor. >Se puede comparar con una dirección en una casa donde cada carpeta o archivo tiene su propio lugar.

Parámetros de consulta: Son valores que se envían al servidor para modificar o personalizar la respuesta. Se añaden al final de la URL usando un signo de interrogación (?)

Fragento: Es una parte opcional que puede dirigir a una sección específica de una página.Se indica con un signo de número (#)

3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Son métodos de petición para indicar la acción que se desea realizar para un recurso determinado. Aunque estos también pueden ser sustantivos, estos métodos de solicitud a veces son llamados HTTP verbs.

  • GET: Solicita una representación de un recurso específico. Las peticiones que usan el método GET sólo deben
    recuperar datos.
  • HEAD: Pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta.
  • POST: se utiliza para enviar una entidad a un recurso en específico, causando a menudo un cambio en el estado o efectos
    secundarios en el servidor.
  • PUT: Reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición.
  • PATCH: Es utilizado para aplicar modificaciones parciales a un recurso.
  • DELETE: Borra un recurso en específico.

5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

El método GET se usa para solicitar información desde un servidor web. Esta solicitud se realiza a través de una URL. Cuando el servidor recibe la solicitud, devuelve la información solicitada. El método GET no cambia los datos en el servidor.

6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Este código indica que la solicitud ha sido exitosa y que el servidor ha devuelto el contenido solicitado sin problemas.

7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

404 es un código de estado HTTP, el cual significa que tu navegador se ha conectado con un servidor, pero que este no es capaz de mostrarte la web que deseas. Puede que también te encuentres con este mismo error, pero con denominaciones como Error 404 not found, Error 404 URL no encontrada, Error 404 página no encontrada o Error 404 Archivo no encontrado, cuando esta problemática se da con un fichero determinado de una web.

El error 404 no es responsabilidad total del desarrollador de resolver, debido a que hay causas de este error de las cuales el desarrollador no puede actuar.

8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

El error 500, también conocido como Internal Server Error (Error Interno del Servidor). es un código de estado HTTP que indica que algo ha salido mal en el servidor web al intentar procesar una solicitud del navegador. Cuando aparece este tipo de error no se proporciona ningún tipo de detalle sobre la causa, lo que complica aún más la identificación de cuál es el problema exacto. La única información certera que tienes es la de saber que el servidor o la propia aplicación informó al navegador de un fallo inesperado, por lo que hay que averiguar que está ocurriendo. El desarrollador es el responsable de arreglar el error, debido a que las principales causas por la cual este error aparece es por fallas en el backend o la base de datos.

Causas: Programación, configuración del servidor, los recursos, software, base de datos, actualizaciones incorrectas, servidor web, limites de tiempo de ejecución, ataques.

10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Las principales diferencias son:

HTML4:

  • Permite la creación de hipervínculos con la (a) etiqueta, conectando diferentes páginas web.
  • Utiliza etiquetas para marcar elementos y contenido, como encabezados ( h1 a h6).
  • Admite la incrustación de imágenes (img), vídeos (video) y audio (audio) para contenido multimedia.
  • Proporciona elementos de formulario como form, input y button para la entrada y el envío de datos por parte del usuario.
  • Etiquetas semánticas como article, section, y nav para una mejor estructura y accesibilidad del documento.
  • No admitía audio ni video sin el uso de compatibilidad con Flash Player
  • Utiliza cookies para almacenar datos temporales
  • no permite que JavaScript se ejecute en el navegador
  • No permite efectos de arrastrar y soltar

HTML5:

  • Se introdujeron nuevos elementos semánticos como header, footer, section y article para mejorar la estructura.
  • Mejora las capacidades multimedia con soporte nativo para elementos de audio y video.
  • Proporciona la localStorageAPI, lo que permite que las aplicaciones web almacenen datos localmente en el dispositivo del usuario.
  • Permite que los sitios web accedan a la ubicación geográfica de un usuario.
  • Utiliza una base de datos SQL para almacenar datos sin conexión.
  • Admite controles de audio y video con el uso de etiquetas audio y video
  • Utiliza base de satos SQL y caché de aplicaciones para almacenar datos sin conexión
  • Permite ejecutar JavaScript en segundo plano Esto gracias a la API de trabajo web de JS en HTML5
  • Permite efectos de arrastrar y soltar

14. ¿Cuál es el ciclo de vida de los sistemas de información?

El ciclo de vida de un sistema de información es un modelo estructurado de gestión de proyectos que describe las fases necesarias para construir un sistema de información, desde el inicio hasta el resultado final. También se refiere a todo el recorrido que realiza un sistema, desde su nacimiento hasta su expiración, abarcando fases extensas. Las fases del ciclo de vida de los sistemas de información son:

  1. Planificación: El analista de sistemas debe centrarse en lo que el sistema pretende lograr y utilizar esa información para encontrar una manera de lograr ese objetivo
  2. Análisis de requerimientos: El analista debe elaborar una lista de todo lo que necesita la empresa para poder desarrollar el mejor sistema teniendo en cuenta los recursos realmente disponibles
  3. Diseño: Esta fase describe como abordar el diseño de la arquitectura del sistema, por ejemplo, las interfaces de usuario, la red de computadoras, la base de datos y las eguridad, que puedan satisfacer los requerimientos y permitan actualizaciones futuras.
  4. Desarrollo: En esta fase, el plano del sistema pasará del modelo a la práctica, a medida que los programadores desarrollen un sistema completamente funcional. Los ingenieros de software escriben el código y van ajustando los tecnologías involucradas en el proyecto.
  5. Prueba: El sistema debe someterse a una inspección exhaustiva en diferentes escenarios. Si se encuentran errores o problemas, el equipo de trabajo deberá alineaarse para resolverlos sin alterar el resto del sistema
  6. Integración y ejecución: Se realiza el primer lanzamiento del sistema. En una situación ideal, la ejecución será tan fluida que no se requerirá ningún esfuerzo adicional cuando ocurra la integración. De ser posible, la integración de un nuevo sistema en una empresa debería ser automática y agil.
  7. Operación y mantenimiento: Aunque las pruebas deberían haber resuelto cualquier problema que pudiera haber surgido, es importante monitorear el nuevo sistema para asegurarse que esté funcionando correctamente. También es importante que el sistema se someta a un mantenimiento frecuente para que pueda seguir funcionando sin problemas.

15. ¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de desarrollo de sistemas de información se centra en las etapas específicas involucradas en la creación de un sistema de información. Aunque las etapas pueden variar según la metodología utilizada, generalmente incluyen:

  1. planificación: El ciclo de desarrollo de sistemas de información se centra en las etapas específicas involucradas en la creación de un sistema de información. Aunque las etapas pueden variar según la metodología utilizada, generalmente incluyen:
  2. Análisis de requerimientos: Se recopilan y documentan los requisitos del sistema, asegurando que se entiendan las necesidades del usuario y los objetivos del negocio.
  3. Diseño: Se elabora un diseño detallado del sistema, que incluye la arquitectura, las interfaces de usuario y la base de datos, asegurando que se cumplan los requisitos.
  4. Desarrollo: En esta fase, se lleva a cabo la programación y la construcción del sistema, donde los desarrolladores crean el software según el diseño especificado.
  5. Pruebas: Se realizan pruebas exhaustivas para identificar y corregir errores antes de que el sistema se implemente en un entorno de producción.
  6. Implementación: Se lanza el sistema al entorno de producción, donde se pone en funcionamiento y se integra con otros sistemas existentes.
  7. Mantenimiento: Después de la implementación, el sistema requiere mantenimiento continuo para corregir errores, realizar actualizaciones y mejorar la funcionalidad según sea necesario.

    2. Aunque Java y JavaScript suenan a dos lenguajes de programación simalares, es todo lo contrario. Mientras que Java es un lenguaje de programación orientado a objetos y se utiliza principalmente para el desarrollo de aplicaciones de escritorio y servidores, JavaScript es un lenguaje de secuencia de comandos del lado del cliente que se utiliza para agregar interactividaad y dinamismo a las páginas web. A pesar de las diferencias mencionados, estos dos lenguajes de programación compartenalgunas simmilitudes. Ambos lenguajes son orientados a objetos, lo que significa que utilizan objetos y clases para organizar y estructurar el código. También se basan en gran medida en la sixtaxsis de C, lo que facilita aprender uno de los dos lenguajes si ya son familiares con el otro.

    3. Métodos Estáticos
    1. Date.now():

      2. Devuelve el valor númerico correspondiente al actual número de milisegundos transcurridos desde el 1 de Enero de 1970, oo:oo:oo UTC,, ignorando los segundos intercalares

    2. Date.parse()

      3. Transforma la cadena que representa una fecha y retorna el número de milisegundos transcurridos desde el 1 de Enero de 1970, ignorando los segundos intercalares. Es muy poco recomendado utilizarlo

    3. Date.UTC()

      4. Acepta los mismos parámetros de la forma extendida del constructor y retorna el numero de milisegundos trsnacurridos desde el 1 de Enero de 1970

      5. Devuelve el mes de la fecha especificada como un número entre 0 (enero) y 11 (diciembre). Se debe usar junto con +1 si deseas obtener el número de mes de forma más intuitiva.

    5. getDate()

      6. Devuelve el día del mes (1-31) de la fecha especificada.

    6. getFullYear()

      7. Devuelve el año de la fecha especificada con cuatro dígitos.

    7. getHours()

      8. Devuelve la hora de la fecha especificada en formato de 24 horas (0-23).

    8. getTime()

      9. Devuelve el número de milisegundos transcurridos desde el 1 de enero de 1970 (Epoch). Se usa frecuentemente para cálculos de tiempo o comparaciones entre fechas.
